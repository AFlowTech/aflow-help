# 设计文档目录

本目录存放aiflow系统的各类设计文档。

## 文档列表

### ERP系统对接方案（主文档）

**文件：** `erp-wms-integration-design.md`

**说明：** 外部ERP系统（如Odoo ERP）对接aiflow流程中心的核心方案文档，包括：

1. **概述**：对接目标、aiflow定位、分阶段实施计划
2. **对接方案概述**：四个主要对接方案的概述和核心要点
3. **方案对比**：不同方案的对比和选择建议
4. **实施建议**：分阶段实施策略和方案选择建议
5. **风险评估**：技术和业务风险分析

**适用场景：**
- 需要快速了解对接方案的整体架构
- 需要选择合适的对接方案
- 需要了解实施策略和风险

**文档结构：**
- 主文档保留核心内容，引用子文档
- 详细技术实现请参考技术原理文档
- 接口定义和任务清单请参考技术详细设计文档

### 技术原理文档

**文件：** `technical-details.md`

**说明：** 详细的技术实现、数据库设计和时序图文档，包括：

1. **基础数据同步方案**：签名算法、时序图、业务逻辑、方案二实现
2. **SSO单点登录对接方案**：OAuth2原理图、时序图、系统改动方案
3. **任务中心统一简易集成方案**：流程定义改造、时序图
4. **全面使用流程中心的表单与流程引擎方案**：流程定义管理、表单同步、ServiceTask推送
5. **技术实现细节**：签名算法实现、数据库设计、接口实现示例、错误处理

**适用场景：**
- 需要了解详细的技术实现原理
- 需要查看时序图和流程图
- 需要了解数据库设计
- 需要参考代码实现示例

**注意：** 接口定义和任务清单已整合到技术详细设计文档中，本文档专注于技术原理和实现细节。

### 技术详细设计文档（整合文档）

**文件：** `integration-technical-design.md`

**说明：** 整合了接口定义和双方任务清单的技术详细设计文档，包括：

**第一部分：技术详细设计**
1. **通用说明**：签名机制（推荐使用Python SDK）、通用请求头、响应格式、错误码
2. **基础数据同步方案**：方案一和方案二的接口定义
3. **SSO单点登录对接方案**：OAuth2流程说明和接口定义
4. **任务中心简易集成方案**：流程发起、待办同步、任务完成回调接口
5. **全面流程引擎集成方案**：流程定义管理、表单同步、审批日志接口

**第二部分：双方任务清单**
- 以表格形式展示各阶段双方需要完成的任务
- 包含接口清单汇总

**适用场景：**
- **推荐双方开发人员优先查阅此文档**
- 需要查看接口的详细定义和参数说明
- 需要了解双方任务分工
- 需要快速了解各阶段的接口和任务

**注意：** 
- 本文档整合了接口定义和任务清单，内容更集中，便于查阅
- 如需了解技术原理和时序图，请参考技术原理文档（`technical-details.md`）
- 文档中已添加对技术原理文档的引用链接，可直接跳转查看

## 文档结构

最终保留三个核心文档：

1. **erp-wms-integration-design.md** - 主方案文档，概述整体对接方案
2. **integration-technical-design.md** - 技术详细设计文档，包含接口定义和任务清单
3. **technical-details.md** - 技术原理文档，包含详细的时序图、数据库设计和实现细节

## 文档维护

设计文档会随着系统演进持续更新，请定期查看最新版本。
